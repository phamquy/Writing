# A Taste of Rust's macro

The term macro refers to a family of features in Rust: **declarative** macros with **macro\_rules!** and three kinds of procedural macros:

* Custom `#[derive]` macros that specify code added with the derive attribute used on structs and enums
* Attribute-like macros that define custom attributes usable on any item
* Function-like macros that look like function calls but operate on the tokens specified as their argument

### The Difference Between Macros and Functions

* Marco run at compile time, function run at executing time
* Function signature need to have number and type of arg, while marco does can take multiple type and number of arg
* Marcro generate code, hard to read understand and maintain
* Function can be defined after being call in code, marco have to be defined before use

### Declarative Macros with `macro_rules!` for General Metaprogramming

Here how `vec`! macro might be implemented:

```rust
#[macro_export]
macro_rules! vec {
    ( $( $x:expr ),* ) => {
        {
            let mut temp_vec = Vec::new();
            $(
                temp_vec.push($x);
            )*
            temp_vec
        }
    };
}
```

* `macro-rules` behave like match statement in Rust code
* `( $( $x:expr ),* )` is matching statement which include
  * `$()` declar the parameter that the match fragement will be assign to, these are type of fragement supported by Rust macro
    * `expr` - an expression (e.g., 1 + 2, foo(), x)
    * `ident` - an identifier (e.g., variable or function name)
    * `ty` - a type (e.g., i32, String, Vec)
    * `pat` - a pattern (e.g., Some(x), \_)
    * `stmt` - a statement
    * `block` - a block of code { ... }
    * `item` - an item (e.g., function, struct, impl)
    * `literal` - a literal value (e.g., 42, "hello")
    * `path` - a path (e.g., std::vec::Vec)
    * `tt` - a token tree (any single token or matched pair of delimiters)

The code generated by macro still need to follow all rust's safety rule:

Example, the following code will fail borrow checker

```rust
macro_rules! manylet {
    ( $( $i:ident ),* = $e:expr ) => {
        $(
            let mut $i = $e;
        )*
    }
}
fn main() {
    let mut s = String::from("A");;
    // This will expanded to: let mut x = s; let mut y = s
    // Which violate single mut reference rule
    manylet!(x, y = s); 
    x.push_str("B");
    println!("{x}{y}");
}

```

To fix it, x, y, s should implement `copy` trait, or we need to explicitly clone it

```rust
macro_rules! manylet {
    ( $( $i:ident ),* = $e:expr ) => {
        $(
            let mut $i = $e;
        )*
    }
}
fn main() {
    let mut s = String::from("A");;
    // This will expanded to: let mut x = s.clone(); let mut y = s.clone()
    manylet!(x, y = s.clone()); 
    x.push_str("B");
    println!("{x}{y}");
}
```

### Procedural Macros for Generating Code from Attributes

* Procedure macro act more like function than match statement
* Take code as input and produce code as output
* Three kinds
  * `derive`
  * attribute-like
  * function-like
* procedule macro will need to be in their own crate
* multiple type of proceduce macro can reside in the same crate

Example:

The following can not be executed in this notebook because it need to be in their own crate

```rust
use proc_macro;

#[proc_macro]
pub fn some_name(input: TokenStream) -> TokenStream {
}
```

#### Write customer `derive` macro

* `derive` ONLY work for structs and enums

What we need to do to define customer derive (to auto implement a trait)

1. Define a trait:

Trait will need to be resided in separate crate that the customer derive macro (limitatio of procedural macro)

```rust
pub trait HelloMacro {
    fn hello_macro();
}
```

2. Create a crate to store the customer derive macro best practice is to store it inside the same folder as the trait, so folder structire like this

```yaml
- hello_macro_trait/
    - src
        - lib.rs <- where u define your trait
    Cargo.toml
    - hello_macro_derive/
        - src
            - lib.rs
        Cargo.toml
```

in the macro crate's Cargo.toml we need to tell compiler that this is a macro crate

```yaml
# in hello_macro_derive/Cargo.toml
[lib]
proc-macro = true
```

```rust
// In hello_macro_derive/src/lib.rs

use proc_macro::TokenStream;
use quote::quote;

#[proc_macro_derive(HelloMacro)]         // Name of the macro : HelloMarco, does not need to be same as trait name, but it's convention
pub fn hello_macro_derive(input: TokenStream) -> TokenStream { // This fn should in generaal same for all custom derive
    // Construct a representation of Rust code as a syntax tree
    // that we can manipulate.
    let ast = syn::parse(input).unwrap();

    // Build the trait implementation.
    impl_hello_macro(&ast)
}

fn impl_hello_macro(ast: &syn::DeriveInput) -> TokenStream {
    let name = &ast.ident; // Getting the symbol identifier (like name of the struct)
    let generated = quote! { // Main generate code 
        impl HelloMacro for #name {
            fn hello_macro() { // We don;t need to bring HelloMacro trait into this scope be cause code enclose in this `{}` is a literal text not rust code
                println!("Hello, Macro! My name is {}!", stringify!(#name));
            }
        }
    };
    generated.into()
}
```

To use it in other crate

```rust
use hello_macro::HelloMacro;
use hello_macro_derive::HelloMacro;

#[derive(HelloMacro)]
struct Pancakes;

fn main() {
    Pancakes::hello_macro();
}
```

#### Attribute-like macros

* Instead of generating code as `derive` macro, it creates new attributes
* More flexible as `derive` ONLY work for structs and enums
* Attributes macro can be applied to other: items, also functions

Example muse in web application for routing

```rust
#[route(GET, "/")]
fn index() {
    ...
}
```

`route` macro would be defined as

```rust
#[proc_macro_attribute]
pub fn route(attr: TokenStream, item: TokenStream) -> TokenStream {...}
```

**Full Implementation Example:**

```rust
// In your proc macro crate (e.g., route_macro/src/lib.rs)
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, ItemFn, LitStr, Ident};

#[proc_macro_attribute]
pub fn route(attr: TokenStream, item: TokenStream) -> TokenStream {
    // Parse the attribute arguments: GET, "/"
    let attr_args = parse_macro_input!(attr as syn::AttributeArgs);
    
    // Parse the function that the attribute is applied to
    let input_fn = parse_macro_input!(item as ItemFn);
    
    // Extract HTTP method (GET) and path ("/")
    let method = &attr_args[0]; // First arg: GET
    let path = &attr_args[1];   // Second arg: "/"
    
    // Extract parts of the function for reconstruction
    let fn_name = &input_fn.sig.ident;     // Function name (e.g., "index")
    let fn_block = &input_fn.block;        // Function body { ... }
    let fn_vis = &input_fn.vis;            // Visibility (pub, pub(crate), or private)
    let fn_sig = &input_fn.sig;            // Full function signature (name, params, return type)
    
    // Generate the wrapped function with route registration
    let expanded = quote! {
        #fn_vis #fn_sig {
            #fn_block
        }
        
        // Auto-register this route with the web framework
        inventory::submit! {
            Route {
                method: Method::#method,
                path: #path,
                handler: #fn_name,
            }
        }
    };
    
    TokenStream::from(expanded)
}
```

**Simpler example that just prints the route info:**

```rust
#[proc_macro_attribute]
pub fn route(attr: TokenStream, item: TokenStream) -> TokenStream {
    // Parse attributes
    let attr_str = attr.to_string();
    
    // Parse the function
    let input_fn = parse_macro_input!(item as ItemFn);
    let fn_name = &input_fn.sig.ident;
    
    // Keep the original function and add route info
    let expanded = quote! {
        #input_fn
        
        // Print route registration at compile time
        const _: () = {
            println!("Registered route: {} -> {}", #attr_str, stringify!(#fn_name));
        };
    };
    
    TokenStream::from(expanded)
}
```

**Usage:**

```rust
#[route(GET, "/")]
fn index() -> String {
    "Hello, World!".to_string()
}

#[route(POST, "/users")]
fn create_user() -> String {
    "User created".to_string()
}
```

The macro receives:

* `attr`: `GET, "/"` as TokenStream
* `item`: The entire `fn index() {...}` as TokenStream

It then generates code that preserves the function and adds route registration logic.

#### Function-Like macro

* Function-like macros define macros that look like function calls.
* Similar to `macro_rules!`, can take unknown number of args
* Unlike `macro_rules!` (only matching expression), it take `TokenStream` and manipulates it

Example

```rust
let sql = sql!(SELECT * FROM posts WHERE id=1);
```

Macro would be defined as

```rust
#[proc_macro]
pub fn sql(input: TokenStream) -> TokenStream {...}
```

**Full Implementation Example:**

```rust
// In your proc macro crate (e.g., sql_macro/src/lib.rs)
use proc_macro::TokenStream;
use quote::quote;

#[proc_macro]
pub fn sql(input: TokenStream) -> TokenStream {
    // Convert the input tokens to a string
    let sql_query = input.to_string();
    
    // Basic validation: check if it's a SELECT query
    let query_upper = sql_query.to_uppercase();
    if !query_upper.starts_with("SELECT") {
        panic!("Only SELECT queries are supported!");
    }
    
    // Generate code that returns the SQL string
    let expanded = quote! {
        {
            // Compile-time SQL validation could happen here
            let query = #sql_query;
            query
        }
    };
    
    TokenStream::from(expanded)
}
```

**More advanced example with compile-time validation:**

```rust
use proc_macro::TokenStream;
use quote::quote;
use syn::parse::{Parse, ParseStream};
use syn::{parse_macro_input, Ident, Token};

// Define a custom parser for SQL-like syntax
struct SqlQuery {
    tokens: Vec<String>,
}

impl Parse for SqlQuery {
    fn parse(input: ParseStream) -> syn::Result<Self> {
        let mut tokens = Vec::new();
        
        // Collect all tokens as strings
        while !input.is_empty() {
            let token = input.parse::<proc_macro2::TokenTree>()?;
            tokens.push(token.to_string());
        }
        
        Ok(SqlQuery { tokens })
    }
}

#[proc_macro]
pub fn sql(input: TokenStream) -> TokenStream {
    let sql_query = parse_macro_input!(input as SqlQuery);
    
    // Join tokens into SQL string
    let query_str = sql_query.tokens.join(" ");
    
    // Perform compile-time validation
    validate_sql(&query_str);
    
    // Generate a struct that holds the query
    let expanded = quote! {
        {
            struct CompiledQuery {
                query: &'static str,
            }
            
            impl CompiledQuery {
                fn execute(&self) -> Result<Vec<Row>, SqlError> {
                    // Execute the query using your database library
                    execute_query(self.query)
                }
            }
            
            CompiledQuery {
                query: #query_str,
            }
        }
    };
    
    TokenStream::from(expanded)
}

fn validate_sql(query: &str) {
    let upper = query.to_uppercase();
    
    // Basic validation rules
    if !upper.starts_with("SELECT") && 
       !upper.starts_with("INSERT") && 
       !upper.starts_with("UPDATE") && 
       !upper.starts_with("DELETE") {
        panic!("Invalid SQL query: must start with SELECT, INSERT, UPDATE, or DELETE");
    }
    
    // Check for balanced parentheses
    let open = query.matches('(').count();
    let close = query.matches(')').count();
    if open != close {
        panic!("SQL syntax error: unbalanced parentheses");
    }
    
    // Additional validation logic...
}
```

**Real-world inspired example (similar to SQLx):**

```rust
use proc_macro::TokenStream;
use quote::quote;

#[proc_macro]
pub fn sql(input: TokenStream) -> TokenStream {
    let query_str = input.to_string();
    
    // Parse and extract column names from SELECT query
    let columns = extract_columns(&query_str);
    
    // Generate a strongly-typed result struct
    let struct_fields = columns.iter().map(|col| {
        quote! { pub #col: String }
    });
    
    let expanded = quote! {
        {
            // Generate a type-safe struct for query results
            struct QueryResult {
                #(#struct_fields),*
            }
            
            // Return the query and result type
            (#query_str, std::marker::PhantomData::<QueryResult>)
        }
    };
    
    TokenStream::from(expanded)
}

fn extract_columns(query: &str) -> Vec<String> {
    // Simplified column extraction
```

**Key differences from `macro_rules!`:**

* Takes arbitrary tokens as `TokenStream`, not pattern-matched
* Can implement custom parsing logic
* Can perform complex compile-time validation
* Can generate complex struct/type definitions
* Requires separate proc-macro crate
